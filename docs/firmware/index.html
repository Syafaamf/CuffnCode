Dokumentasi Firmware

Arsitektur Firmware

Firmware CuffnCode dibangun di atas STM32 HAL (Hardware Abstraction Layer) dengan arsitektur berbasis state machine. Setiap modul hardware memiliki driver sendiri dengan API yang bersih dan terdokumentasi.

Layers

LayerKomponenFungsi
Applicationmain.c, pressure.cState machine, algoritma oscillometric
Driveradc.c, pump.c, valve.c, uart.cHardware abstraction
Signal Processingfilter.cFIR, IIR, moving average, notch filter
HALSTM32F4xx_HAL_DriverSTM32 Hardware Abstraction Layer
CMSISCMSIS-CoreCortex-M4 CMSIS

State Machine

Sistem dikontrol oleh finite state machine dengan 6 state: 

IDLE → (CMD 's') → INFLATING → (target tercapai) → MEASURING
MEASURING → (tekanan < 20) → COMPLETE → IDLE
ANY → (error) → ERROR → (CMD 'r') → IDLE
IDLE → (CMD 'd') → DUMP → IDLE
StateDeskripsiAksi
STATE_IDLEMode standbyLED slow blink 1Hz, tunggu command UART
STATE_INFLATINGCuff mengembangPump ON sampai 180mmHg, monitor tekanan
STATE_MEASURINGMengukur tekananValve controlled deflation, sampling 1kHz
STATE_COMPLETEPengukuran selesaiKirim hasil via UART (JSON + CSV), LED flash
STATE_ERRORError terdeteksiEmergency deflation, LED SOS pattern
STATE_DUMPDump data mentahKirim raw buffer via UART (CSV format)

Timing

Defulasi + Sampling~30-45 detik3-5 mmHg/s, dari 180 ke 20 mmHg

API Documentation

ADC Driver (adc.c/h)

Driver untuk ADC1 dengan DMA double buffer dan timer trigger @1kHz.

Fungsi Utama

FungsiDeskripsi
ADC_Init()Inisialisasi parameter kalibrasi
ADC_Calibrate()Kalibrasi offset ADC (100 sampel rata-rata)
ADC_GetRawValue()Mendapatkan nilai ADC mentah (0-4095)
ADC_GetVoltage()Konversi ke tegangan (V)
ADC_GetPressure_mmHg()Konversi ke tekanan (mmHg)
ADC_StartSampling()Mulai sampling DMA + timer
ADC_StopSampling()Hentikan sampling

Konversi ADC ke mmHg

Vadc = (raw × 3.3V) / 4095
PmmHg = (Vadc - 1.5V) × 300 / (3.3V - 1.5V)

Build Instructions

Prerequisites

  • ARM GCC Toolchain (arm-none-eabi-gcc)
  • Make (GNU Make)
  • STM32CubeProgrammer atau OpenOCD untuk flashing
  • STM32CubeMX (opsional, untuk regenerasi kode HAL)

Build

cd Firmware/
make all

Output:

  • build/cuffncode.elf - ELF file (debug)
  • build/cuffncode.bin - Binary file (flashing)
  • build/cuffncode.hex - Intel HEX file
  • build/cuffncode.map - Memory map

Flash via ST-Link

make flash          # Via STM32_Programmer_CLI
make flash-openocd  # Via OpenOCD

Flash via STM32CubeProgrammer (Manual)

STM32_Programmer_CLI -c port=SWD -w build/cuffncode.bin 0x08000000 -v -rst

Clean

make clean
Catatan: Pastikan ST-Link/V2 terhubung ke pin SWDIO dan SWCLK Black Pill sebelum flashing.

Algoritma Oscillometric

Metode oscillometric adalah standar industri untuk pengukur tekanan darah digital otomatis. Algoritma ini menganalisis osilasi dinding arteri yang ditransmisikan ke cuff selama deflasi.

Prinsip Kerja

  1. Inflasi: Cuff dikembangkan hingga tekanan ~180 mmHg (di atas tekanan sistolik normal)
  2. Defulasi Bertahap: Katup membuka secara perlahan (~3-5 mmHg/s)
  3. Sampling: ADC membaca tekanan cuff dan osilasi @1000 Hz
  4. Filtering: Band-pass filter 0.5-5Hz mengekstrak osilasi dari sinyal DC tekanan
  5. Envelope: Amplitudo puncak osilasi diplot terhadap tekanan cuff
  6. MAP: Tekanan pada amplitudo osilasi maksimum
  7. SBP: Tekanan pada 55% amplitudo maksimum (sisi tekanan tinggi)
  8. DBP: Tekanan pada 75% amplitudo maksimum (sisi tekanan rendah)
  9. HR: Frekuensi osilasi = heart rate

Parameter Algoritma

ParameterNilaiKeterangan
Sampling Rate1000 HzFrekuensi ADC + timer trigger
Band-pass Filter0.5 - 5 HzRentang frekuensi osilasi arteri
SBP Ratio0.55 × maxPersentase amplitude untuk SBP
DBP Ratio0.75 × maxPersentase amplitude untuk DBP
Peak Detection Window20 sampelWindow untuk deteksi puncak
Envelope Smoothing3 titikMoving average envelope
Min Oscillation0.01Threshold minimum osilasi
Referensi: Rasio SBP (0.55) dan DBP (0.75) adalah nilai umum untuk algoritma oscillometric. Nilai ini dapat disesuaikan berdasarkan kalibrasi dengan alat referensi.
